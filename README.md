# Інструкція з розгортання Canteen ERP

Цей посібник допоможе вам запустити проект на новому комп'ютері після клонування репозиторію.

## 1. Попередні вимоги
Переконайтеся, що на вашому комп'ютері встановлені:
- **Python 3.10+**
- **Docker Desktop** (для бази даних PostgreSQL)
- **Git**

---

## 2. Клонування та налаштування оточення

Відкрийте термінал у папці з проектом:

```bash
# 1. Створення віртуального оточення
python -m venv venv

# 2. Активація оточення (Windows)
.\venv\Scripts\activate

# 3. Встановлення залежностей
pip install -r requirements.txt
```

---

## 3. Запуск бази даних (Docker)

Проект використовує PostgreSQL у Docker. Налаштування вже прописані у `docker-compose.yml`.

```bash
# Запуск контейнера з БД у фоновому режимі
docker compose up -d
```

*Примітка: База даних буде доступна на `localhost:5432` з логіном `postgres` та паролем `433456`.*

---

## 4. Налаштування Django

Після того, як база даних запустилася, потрібно підготувати структуру таблиць та створити адміністратора.

```bash
# 1. Пристосування міграцій (створення таблиць)
python manage.py migrate

# 2. Створення суперкористувача (Адміністратора)
# Дотримуйтесь інструкцій у терміналі (логін, email, пароль)
python manage.py createsuperuser

# 3. (Опціонально) Перевірка системи
python manage.py check
```

---

## 5. Запуск проекту

Тепер можна запускати сервер розробки:

```bash
python manage.py runserver
```

Проект буде доступний за адресою: [http://127.0.0.1:8000/](http://127.0.0.1:8000/)

- **Панель адміністратора:** `/admin/`
- **Dashboard адміна їдальні:** `/dashboard/`
- **POS-термінал касира:** `/pos/`
- **Меню клієнта:** `/client/`

---

## 6. Тестування

Щоб переконатися, що все працює коректно, запустіть тести:

```bash
pytest
```

---

## Можливі проблеми (Troubleshooting)

1. **Помилка підключення до БД**: Переконайтеся, що Docker контейнер `pos_postgres` запущений (`docker ps`).
2. **Помилка psycopg2 на Windows**: Якщо виникають проблеми з кириличними шляхами, у `settings.py` активовано обхідний шлях через `PGPASSFILE`. Переконайтеся, що у вас є права на запис у корінь диска `C:\` або змініть шлях у налаштуваннях.
3. **Статичні файли (CSS/JS)**: Якщо стилі не відображаються, виконайте `python manage.py collectstatic`.
